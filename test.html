ts

  changeDetection: ChangeDetectionStrategy.OnPush

  • Load feature modules only when the user navigates to them.

ts

  path: 'admin',
  loadChildren: () => import('./admin/admin.routes').then(m => m.routes),
}

üîπ 3. Standalone Components and Declarative Routing (Angular 15+)

  • Use standalone components to reduce NgModule overhead.

  • Declarative routing improves clarity and performance.

üîπ 4. TrackBy in ngFor

  • Prevent re-rendering of unchanged DOM elements.

html
<li *ngFor="let item of items; trackBy: trackByFn">{{ item.name }}</li>

üîπ 5. Use Pure Pipes

  • Ensure pipes are pure by default to avoid unnecessary recalculations.

üîπ 6. Avoid Memory Leaks

  • Unsubscribe from Observables using takeUntil, AsyncPipe, or ngOnDestroy.

html
<p>{{ user$ | async }}</p> <!-- AsyncPipe auto-unsubscribes -->

  • Use Angular CLI's image optimizer or external tools.

üîπ 8. Preload Only Important Modules

  • Use PreloadAllModules or custom strategies for smarter preloading.

üîπ 9. Use Efficient DOM Structures

  • Avoid deep component trees.

üîπ 10. Server-Side Rendering (Angular Universal)
